### 完整处理数据流

```
视频文件 (任意格式)
  │
  ▼  VideoReader.frames() — 逐帧 yield, 绝不全量加载
  │
  ├─→ DanceTracker.track(frame)
  │     │
  │     ├─ _detect():
  │     │   model.predict(src, conf=0.3, iou=0.35, classes=[0],
  │     │                 retina_masks=True, imgsz=1024)
  │     │   → mask_bool = prob > 0.25         ← 掩码二值化阈值
  │     │   → 过滤 min_mask_area < 2000px     ← 防杯子误检
  │     │
  │     ├─ _associate():
  │     │   IoU 贪心匹配 (threshold=0.3)
  │     │   → 未匹配 → lost_tracks (缓冲 60 帧)
  │     │   → 恢复 lost / 创建新 track (≤ stable_count)
  │     │
  │     ├─ _build_results():
  │     │   active_tracks + ghost_tracks (lost ≤ 3帧)
  │     │
  │     └─ _impute_masks():
  │         面积比率检测 → BBox 对齐平移融合
  │
  ▼  process_frame_effects(frame, track_results)
  │     │
  │     ├─ calculate_depth_order() — 脚部 Y 坐标排序
  │     ├─ _morphology_open()       — 5×5 开运算去毛刺
  │     ├─ _prepare_body_mask()     — 贴骨转换 (expand=0)
  │     ├─ TemporalMaskCache        — EMA 时序平滑 (8帧)
  │     └─ apply_shadow_outline_effect()
  │           ├─ body 填充: solid / gradient / blur
  │           └─ edge 边框: 椭圆核膨胀 + 高斯模糊抗锯齿
  │
  ▼  VideoWriter.write(result) — 临时 mp4v 视频
  │
  ▼  merge_audio_with_moviepy() — libx264 + AAC → 最终成品
```

### 3.3 track() — 公开接口

```python
def track(self, frame: np.ndarray) -> List[TrackResult]:
    detections = self._detect(frame)                  # Step 1: YOLO
    self._associate(detections, frame_shape)           # Step 2: 关联
    self._purge_expired()                              # Step 3: 清理
    results = self._build_results(frame_shape)         # Step 4: 输出
    results = self._impute_masks(results, frame_shape) # Step 5: 修补
    self._frame_idx += 1
    return results
```

### 3.4 _detect() — YOLO 推理

```python
def _detect(self, frame) -> List[dict]:
    results = self.model.predict(
        source=frame,
        conf=0.3,              # 置信度阈值
        iou=0.35,              # NMS IoU
        classes=[0],           # 仅 person
        retina_masks=True,     # 原生分辨率 mask
        imgsz=1024,            # 推理分辨率
        device=self._device,
        verbose=False,
        half=self.config.half,
    )
    # 遍历每个检测结果:
    for each box:
        m = result.masks.data[i].cpu().numpy()
        mask_bool = m > 0.25           # ← ★ 掩码二值化阈值
        if mask_bool.shape != (h,w):
            mask_bool = cv2_resize_mask(mask_bool, w, h)
        if np.count_nonzero(mask_bool) < 2000:
            continue                    # ← ★ 过滤小目标误检
        detections.append({"bbox": ..., "conf": ..., "mask": mask_bool})
```

**关键参数说明**：

| 参数 | 值 | 作用 |
|------|-----|------|
| `retina_masks=True` | — | 输出与输入帧同分辨率的 mask，不做上采样 |
| `imgsz=1024` | — | YOLO 内部推理分辨率 |
| `m > 0.25` | 掩码二值化阈值 | 0.25 = 低阈值捕获手臂/手掌低置信度区域 |
| `min_mask_area=2000` | 防误检 | 720p 下真人 mask ≥ 50000px，杯子 < 500px |

**retina_masks 机制**：ultralytics 默认 `retina_masks=False` 时，mask 在模型内部特征图尺寸 (如 128×128) 上计算，然后双线性上采样到原图。`retina_masks=True` 直接在原图分辨率上做 mask head 推理，保留细长结构的像素级精度。

### 3.6 _build_results() — 结果输出 (含幽灵帧)

```python
def _build_results(self, frame_shape):
    results = []
    # 活跃轨迹
    for tid, t in self._active_tracks.items():
        results.append(_make_track_result(tid, t.last_mask, ...))

    # 幽灵轨迹: lost ≤ ghost_frames(3), 复用历史 mask
    for tid, t in self._lost_tracks.items():
        if t.lost_frames <= 3:
            ghost_conf = t.last_conf * (0.9 ** t.lost_frames)
            results.append(_make_track_result(tid, t.last_mask, ...))
    return results
```

### 3.7 _impute_masks() — 掩码质检 + BBox 对齐修补

```
for each TrackResult:
  ├─ 无缓存 → 初始化 last_valid_masks
  ├─ mask 为空 → 复用上一帧
  └─ 面积比率检测:
      area_ratio = current_area / prev_area
      if ratio < 0.7:        # 面积暴跌 >30%
          bbox_ratio = curr_bbox_area / prev_bbox_area
          if bbox_ratio >= 0.85:    # BBox 稳定 → 确认是 mask 残缺
              dx, dy = 当前bbox中心 - 历史bbox中心
              aligned_prev = cv2.warpAffine(prev_mask, 平移矩阵)
              repaired = current | aligned_prev   # Bitwise OR
      不更新缓存 (保留健康 mask)
      else:
          更新缓存
```

`_translate_mask()` 使用 `cv2.warpAffine` + `INTER_NEAREST` 实现高效平移。

## 4. src/effects.py — 特效渲染模块

### 4.1 渲染管线

```
原始 YOLO mask (bool, retina_masks 原生分辨率)
  │
  ├─ _morphology_open(mask, 5×5)     → uint8 清洁 mask
  ├─ _prepare_body_mask(opened, 0)   → float32 贴骨转换
  ├─ TemporalMaskCache.update/EMA    → 时序平滑
  │
  ├─ 阴影/填充渲染:
  │   ├─ solid:    纯色 alpha 混合 → result = src*(1-α) + color*α
  │   ├─ gradient: 竖向线性渐变 + mask 裁切
  │   └─ blur:     GaussianBlur(31px) + mask 区域替换
  │
  └─ 边框渲染:
      _extract_edge_mask → dilate(3×3椭圆核) - body → GaussianBlur(7,1.0)
      → alpha 混合自定义颜色
```

### 4.3 EMA 公式

```
EMA_t = 0.5 × EMA_{t-1} + 0.5 × X_t     (decay=0.5 in TemporalMaskCache)
实际使用 decay=0.4 (来自 process_frame_effects):
EMA_t = 0.4 × EMA_{t-1} + 0.6 × X_t

权重分布:  t=60%  t-1=24%  t-2=9.6%  t-3=3.8%  ...
```

## 9. 肢体截断问题分析

### 症状

舞者手臂/手掌向外伸展或快速挥舞时，主躯干完美打码，但延伸的肢体暴露在黑影外。

### 根因诊断 (按影响力排序)

#### 根因 A：形态学开运算吃掉细长结构 (最可疑)

`effects.py:43-46`:
```python
opened = cv2.morphologyEx(mask_u8, cv2.MORPH_OPEN, kernel=5×5 ones, iterations=1)
```

MORPH_OPEN = **先腐蚀(erode)再膨胀(dilate)**。

- 5×5 腐蚀会把宽度 < 5px 的结构完全抹掉
- 舞蹈中的**手指、手腕、伸展手臂**经过 1024px 推理后可能只有 3-8px 宽
- 腐蚀阶段直接删除了这些细长区域
- 后续膨胀只能恢复剩余的主体，细长结构永远丢失

**这是最可能导致"缺胳膊"的根因**——不是 YOLO 没检测到，而是 YOLO 输出的 mask 在进入特效渲染前被开运算吃掉了。

#### 根因 B：BBox 截断效应

YOLO 先预测 BBox 再算 mask。如果 BBox 没完全包住伸展的手臂，mask 头只会在 BBox 内计算，BBox 外的像素直接丢弃。

#### 根因 C：训练集偏差

COCO 数据集的人体标注对精细手指/伸展肢体覆盖不足，模型本身对这类区域的概率输出偏低（0.2-0.4），即使阈值降到 0.25 也不够。

### 修复优先级

| 优先级 | 修复 | 预期效果 |
|--------|------|---------|
| P0 | 减小开运算核 5→2，或用闭运算替代 | 保留细长肢体不被腐蚀 |
| P1 | imgsz 1024→1280 | 提高手指/手腕的像素分辨率 |
| P2 | BBox 膨胀 20% 后重新推理 mask | 兜底截断 |
