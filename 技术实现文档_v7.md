## 2. 总数据流

```
VideoReader.frames() 逐帧读取
  │
  ▼
DanceTracker.track(frame)
  ├── model.track(persist=True, tracker="botsort.yaml", retina_masks=True, imgsz=1280)
  ├── prob(m) → alpha: clip((m - 0.05) / 0.35, 0, 1)
  │         → alpha > 0.8 → 硬置 1.0
  │         → m > 0.05 且 alpha < 0.1 → 抬底到 0.1
  ├── alpha.sum() < 800 → 过滤跳过
  ├── BBox 膨胀: 水平 10%, 垂直 5%
  ├── foot_y: alpha 行权重加权中心
  ├── 正常检测 → 存入 _fallback_alphas 缓存
  └── 漏检 ID → 缓存 ×0.85 衰减补位 (sum < 400 则彻底删除)
  │
  ▼  List[TrackResult]  (mask = float32 alpha [0,1])
  │
  ▼
process_frame_effects(frame, track_results, ...)
  ├── target_ids 过滤
  ├── calculate_depth_order() → 首帧冻结 foot_y 排序 (无 EMA)
  ├── TemporalMaskCache.update(): EMA = 0.3*old + 0.7*current
  ├── get_blended_body(): GaussianBlur(3×3, σ=0.8)
  ├── get_blended_edge(): GaussianBlur(5×5, σ=1.0) → _extract_edge_alpha()
  │     └── alpha>0.3 二值化 → findContours(RETR_EXTERNAL) → dilate - outer → GaussianBlur
  ├── apply_shadow_outline_effect():
  │     Pass 1: 远→近 画填充 (solid / gradient / blur)
  │     Pass 2: 统一画所有白边 (最上层)
  └── cleanup_stale(): 清理已消失 ID 的 EMA 缓存
  │
  ▼  BGR uint8 ndarray
  │
  ▼
VideoWriter.write(frame) 逐帧写出 (mp4v)
  │
  ▼
merge_audio_with_moviepy(): libx264 + AAC 最终合成
```

**逐检测处理流程**:

```
for i in range(len(result.boxes)):
    1. 取 track_id: result.boxes.id[i].item()
       → 加入 seen_ids

    2. 取 bbox: result.boxes.xyxy[i] → (x1, y1, x2, y2) int

    3. 取 conf: result.boxes.conf[i]

    4. Soft Alpha 转换 (3 步):
       m = result.masks.data[i]  (先 resize 到原帧尺寸如果不等)
       alpha = np.clip((m - 0.05) / 0.35, 0.0, 1.0)    # 步骤1: 线性映射
       alpha = np.where(alpha > 0.8, 1.0, alpha)         # 步骤2: 高值硬截断
       alpha = np.where(m > 0.05, np.maximum(alpha, 0.1), alpha)  # 步骤3: 低值抬底
```

**alpha 转换映射表**:

| raw prob (m) | 步骤1 clip | 步骤2 截断 | 步骤3 抬底 | 最终 alpha |
|-------------|-----------|-----------|-----------|-----------|
| 0.00 | 0.00 | 0.00 | 0.00 (m≤0.05) | **0.00** |
| 0.05 | 0.00 | 0.00 | 0.00 | **0.00** |
| 0.10 | 0.14 | 0.14 | max(0.14, 0.1)=0.14 | **0.14** |
| 0.15 | 0.29 | 0.29 | max(0.29, 0.1)=0.29 | **0.29** |
| 0.20 | 0.43 | 0.43 | max(0.43, 0.1)=0.43 | **0.43** |
| 0.30 | 0.71 | 0.71 | max(0.71, 0.1)=0.71 | **0.71** |
| 0.35 | 0.86 | 1.00 | 1.00 | **1.00** |
| 0.40 | 1.00 | 1.00 | 1.00 | **1.00** |

步骤3 的含义: 只要 `m > 0.05`，alpha 至少 0.1（确保边缘不消失）

```
    4. 过滤: if alpha.sum() < config.min_alpha_sum (800): continue

    5. BBox 膨胀:
       bw, bh = x2-x1, y2-y1
       expand_w = int(bw * 0.10)    # 水平 10%
       expand_h = int(bh * 0.05)    # 垂直 5%
       clamp 到图像边界

    6. foot_y 计算:
       if alpha.sum() > 100:
           row_weights = alpha.sum(axis=1)
           ys = where(row_weights > 0.01)
           foot_y = ys.max()   # alpha 加权底部
       else:
           foot_y = y2         # 退化到 bbox 底部

    7. 更新缓存: self._fallback_alphas[track_id] = alpha.copy()

    8. 创建 TrackResult 并加入结果列表
```

**时序 Fallback** (在所有检测处理完后):

```python
for tid, cached_alpha in list(self._fallback_alphas.items()):
    if tid not in seen_ids:                    # 当前帧未检测到此 ID
        decayed = cached_alpha * 0.85          # 衰减
        if decayed.sum() > 400:                # 还有足够 alpha
            # 创建一个 TrackResult 补位
            # bbox = (0, 0, w, h) 全图
            # confidence = 0.3
            # foot_y = argmax(decayed.sum(axis=1))
            self._fallback_alphas[tid] = decayed
        else:
            del self._fallback_alphas[tid]      # 彻底丢失
```

Fallback 的特点:
- 使用全图 bbox `(0, 0, w, h)`，因为不知道具体位置
- 衰减因子 0.85 每帧递减
- sum < 400 的硬删除门槛 (而非靠帧数 ghost_frames)
- 注意: 并不检查 ghost_frames 参数，只靠 sum 阈值

**最终**: `track_results.sort(key=lambda t: t.track_id)`

## 5. src/effects.py — 特效 & 深度排序模块

### 5.1 白边提取 `_extract_edge_alpha()`

```python
def _extract_edge_alpha(alpha: np.ndarray, edge_width: int = 3) -> np.ndarray:
```

**不是 pure soft 方案。仍依赖二值化 + 轮廓提取**:

```
1. hard_u8 = (alpha > 0.3) × 255            ← 硬阈值 0.3 二值化
2. contours = findContours(hard_u8, RETR_EXTERNAL, CHAIN_APPROX_SIMPLE)
   → 只取最外层轮廓，忽略内部洞
3. 若无轮廓 → 返回全零
4. outer = 全零画布
   drawContours(outer, contours, -1, 255, thickness=FILLED)   ← 填实外轮廓
5. kernel = MORPH_ELLIPSE(edge_width, edge_width)
   dilated = dilate(outer, kernel)
6. edge = (dilated - outer) / 255.0          ← 膨胀差 = 白边
7. return GaussianBlur(edge, (3,3), sigmaX=0.5)  ← 抗锯齿
```

**关键点**:
- 白边基于 `alpha > 0.3` 的硬二值化区域，而非 soft alpha 梯度
- `RETR_EXTERNAL` 忽略内部波动（如手指间缝隙），只提取最外层轮廓
- 白边只能是整数像素宽度 (通过 dilate kernel size 控制)
- 这实际上是 v5 的旧方案，从未改为 v6 文档声明的 `GaussianBlur(alpha) - alpha`

### 5.2 EMA 时序缓存 `TemporalMaskCache`

```python
class TemporalMaskCache:
    def __init__(self, window_size=5, ema_decay=0.3):
        # window_size 参数定义了但从未使用
        self.ema_decay = ema_decay
        self._cache: Dict[int, np.ndarray] = {}

    def update(self, track_id: int, alpha: np.ndarray, frame_idx: int):
        # frame_idx 参数定义了但从未使用
        if track_id not in self._cache:
            self._cache[track_id] = alpha.copy()
            return
        # EMA 公式: new = 0.3*old + 0.7*current
        self._cache[track_id] = (ema_decay * self._cache[track_id]
                                  + (1.0 - ema_decay) * alpha)

    def get_blended_body(self, track_id: int) -> Optional[np.ndarray]:
        alpha = self._cache.get(track_id)
        if alpha is None: return None
        return np.clip(GaussianBlur(alpha, (3,3), sigmaX=0.8), 0.0, 1.0)

    def get_blended_edge(self, track_id: int, edge_width: int = 3) -> Optional[np.ndarray]:
        alpha = self._cache.get(track_id)
        if alpha is None: return None
        smoothed = GaussianBlur(alpha, (5,5), sigmaX=1.0)   # 先做更大平滑
        return _extract_edge_alpha(smoothed, edge_width)     # 再提取白边

    def cleanup_stale(self, active_ids: set):
        # 删除 _cache 中不在 active_ids 里的 track_id

    def reset(self):
        self._cache.clear()
```

**EMA 公式**: `cache_new = ema_decay × cache_old + (1 - ema_decay) × alpha_current`

默认 `ema_decay=0.3`，即 **30% 历史 + 70% 当前**，历史权重较低（对变化敏感，但也更抖）。

**未使用的参数**: `window_size` (构造时传入但无引用), `frame_idx` (update 接收但无引用)

### 5.3 深度排序 `calculate_depth_order()`

```python
def calculate_depth_order(
    track_results: List,
    temporal_window: int = 5,              # 定义了但未使用 (除用作 max 计算)
    smooth_history: Optional[dict] = None,  # 接收了但仅透传不修改
) -> Tuple[List[int], dict]:
```

**实现: 首帧冻结，不做 EMA，不做死区**:

```python
frozen_order = getattr(calculate_depth_order, "_frozen_order", None)
current_ids = [t.track_id for t in track_results]

if frozen_order is None:                                    # 首次调用
    sorted_tracks = sorted(track_results, key=lambda t: t.foot_y)
    frozen_order = [t.track_id for t in sorted_tracks]      # foot_y 从小到大 (远→近)
    calculate_depth_order._frozen_order = frozen_order       # 永久冻结

# 之后每帧: 保持冻结顺序，只过滤/追加
ordered = [tid for tid in frozen_order if tid in current_ids]
for tid in current_ids:
    if tid not in ordered:
        ordered.append(tid)    # 新 ID 追加到末尾 = 画在最上面

return ordered, (smooth_history or {})
```

**深度决定因子**: `foot_y` (alpha 加权底部坐标)。foot_y 越小的人越先画 (越远)。
**冻结时机**: 第一次调用此函数时确定，对 CLI 模式就是第一帧，对 API 关键帧就是 analyze 那一帧。
**新增人物**: 直接 append 到冻结顺序末尾，画在已有所有人物之上。
**smooth_history**: 接收后完全不修改，仅确保返回值不为 None。

### 5.5 特效渲染核心 `apply_shadow_outline_effect()`

```python
def apply_shadow_outline_effect(
    frame: np.ndarray,
    depth_order: List[int],
    track_id_to_idx: dict,
    track_results: List,
    temporal_cache: TemporalMaskCache,
    frame_idx: int,
    dilate_kernel_size: int = 3,
    fill_mode: str = "solid",          # "solid" | "gradient" | "blur"
    fill_color: str = "#000000",
    border_color: str = "#FFFFFF",
    opacity: float = 1.0,
    target_ids: Optional[List[int]] = None,  # 定义了但未使用
) -> np.ndarray:
```

**两阶段渲染**:

**Pass 1 — 远→近画填充**:

```
for tid in depth_order:             # 远的先画
    1. 取 TrackResult.mask (float32 alpha)
    2. temporal_cache.update(tid, alpha, frame_idx)
    3. blended_body = temporal_cache.get_blended_body(tid)
       → GaussianBlur(cached_alpha, (3,3), σ=0.8)
    4. blended_edge = temporal_cache.get_blended_edge(tid, dilate_kernel_size)
       → GaussianBlur(cached_alpha, (5,5), σ=1.0) → _extract_edge_alpha()
    5. 收集 (tid, blended_edge) 到 all_edges 列表
    6. 根据 fill_mode 绘制填充:
       - "solid":   result = result*(1-α) + fill_color*α
       - "gradient": result = result*(1-α) + gradient_layer   (顶部到底部减60)
       - "blur":    result = frame*(1-α) + GaussianBlur(frame,31,σ=20)*α
       所有 mode 都乘以 opacity
```

**Pass 2 — 统一画白边**:

```
for _, e in all_edges:              # 按远→近顺序
    if e.max() > 0.001:
        result = result*(1-e) + border_color*e
```

白边画在 Pass 1 所有人的填充之上。因为 all_edges 按 depth_order 顺序 (远→近)，后面的人白边会覆盖前面的人，符合近处优先的直觉。

**关键特征**:
- 不做实例互斥切割，render 时每人 alpha 独立混合
- `target_ids` 参数在函数签名中定义但完全未使用（过滤已经在调用方 `process_frame_effects` 完成）
- `frame_idx` 传给 `temporal_cache.update()` 但实际上 update 内部不用它

```python
    show_progress: bool = True,
    cancel_event = None,          # threading.Event, 支持外部中断
    fill_mode: str = "solid",
    fill_color: str = "#000000",
    border_color: str = "#FFFFFF",
    opacity: float = 1.0,
) -> str:
```

**处理流程**:

```
1. _init_tracker() — 首次调用时懒加载 DanceTracker

2. 读取视频信息 (get_video_info)

3. 初始化:
   smooth_history = {}
   temporal_cache = None   (首次传给 process_frame_effects 时会自动创建)
   dilate_kernel_size = effect_config.get("dilate_kernel_size", 3)
   temporal_window = effect_config.get("temporal_window", 8)

4. 创建临时视频路径: output_path + ".tmp_video.mp4"

5. 主循环 try:
   with VideoReader(input_path) as reader:
   with VideoWriter(tmp_video, fps, size, "mp4v") as writer:
        for frame_idx, frame in reader.frames():
            a. cancel_event 检查 (如果 set 则 break)
            b. track_results = self._tracker.track(frame)
            c. result_frame, smooth_history, temporal_cache =
               process_frame_effects(...)
            d. writer.write(result_frame)
            e. 异常降级: writer.write(frame)  + 打印警告

6. 音频处理:
   if has_audio_stream(input_path):
       merge_audio_with_moviepy(tmp_video, input_path, output_path)
   else:
       os.rename(tmp_video, output_path)

7. finally: 清理临时视频
```

**dilate_kernel_size 读取路径**: `effect_config["dilate_kernel_size"]`, 默认 3。此值传给 `_extract_edge_alpha` 的 `edge_width` 参数，决定白边宽度。

**temporal_window 读取路径**: `effect_config["temporal_window"]`, 默认 8。传给 `process_frame_effects`，最终传给 `calculate_depth_order` 的 `max(3, temporal_window // 2)`。

**注意**: 这些 `effect_config` 键只在 `main.py` 和 `api.py` 构造时显式传入。`config.yaml` 的 `effects.open_kernel_size` 和 `effects.blur_sigma` 虽被 `main.py` 读入 `effect_config`，但在 pipeline 内部没有任何代码消费它们。

## 8. api.py — FastAPI Web 服务

启动: `uvicorn api:app --host 0.0.0.0 --port 8002`

### 8.1 全局状态

```python
TASKS: Dict[str, dict] = {}   # 内存中的任务字典
TASKS_DIR = "data/tasks"       # 每个任务一个子目录
```

### 8.2 API 端点

#### `GET /` — 前端页面
返回内嵌的 SPA HTML (INDEX_HTML)。

#### `DELETE /cleanup/{task_id}` — 资源释放
```python
task = TASKS.pop(task_id, None)
shutil.rmtree(task["video_path"] 上级目录)
```

#### `POST /analyze` — 上传并分析

```
1. 生成 task_id (uuid4 hex 前10位)
2. 保存上传文件到 data/tasks/{task_id}/source{ext}
3. 用 cv2.VideoCapture 取 total_frames 和 fps
4. 扫描 12 帧选最佳:
   - 新开一个 YOLO("yolo11s-seg.pt") scanner (硬编码)
   - 12 个均匀采样位置
   - scanner.predict(frame, classes=[0], conf=0.3, device="cpu", verbose=False)
   - 只统计 boxes 数量 (不跑 tracker, 不提取 mask)
   - 选人数最多的帧为 best_frame
5. 在 best_frame 上跑完整 DanceTracker:
   - TrackerConfig(device="cpu", verbose=False)
   - 获取 track_results (含 mask + track_id)
6. 用默认参数渲染关键帧:
   - _render_one_frame(key_frame, track_results, all_ids, defaults)
   - defaults = {fill_mode:"solid", fill_color:"#000000", border_color:"#FFFFFF",
                  opacity:1.0, thickness:3}
7. 存储到 TASKS[task_id]:
   - video_path, ext, key_frame, track_results, available_ids,
     total_frames, fps, key_frame_idx
8. 返回:
   - task_id, image_base64 (JPEG quality 85),
     available_ids (排序后的 track_id 列表), total_frames, fps
```

#### `POST /preview_frame` — 单帧重渲染

```
1. 校验 task_id
2. 解析 target_ids
3. 调用 _render_one_frame(task["key_frame"], task["track_results"], parsed_ids, params)
   - 使用 TemporalMaskCache(ema_decay=0.0) → 无 EMA，纯当前帧
   - calculate_depth_order 读 _frozen_order → 与 analyze 时的首帧冻结一致
4. 在结果图上标注人物 ID 标签 (bbox 左上角黑底白字)
5. 返回 base64 JPEG
```

**`_render_one_frame()` 的 ID 标注**: 在每个 track 的 bbox 左上画 `ID:N`，不参与 render pass。

#### `POST /preview_snippet` — 3秒片段 (async, 可中断)

```
1. 创建独立的 DanceTracker (不共享追踪状态)
2. 逐帧 track + process_frame_effects
   - TemporalMaskCache(window_size=4, ema_decay=0.4)  ← 与 pipeline 默认不同
   - process_frame_effects 使用传入的参数 (fill_mode, color, thickness, opacity)
3. 每帧前检查 request.is_disconnected()，断开则中止
4. 写出 mp4v 临时视频
5. moviepy: libx264 + AAC 重编码为 snippet.mp4
6. 返回 FileResponse
```

#### `POST /render` — 全片渲染 (async, 可中断)

```
1. cancel_event = threading.Event()
2. 创建 asyncio 后台任务轮询 request.is_disconnected()
3. 构建 DanceAnonymizerPipeline:
   - tracker_config: model="yolo11s-seg.pt", device=用户指定, conf=0.3, verbose=False
   - effect_config: {body_expand_pixels: 0, dilate_kernel_size: max(1, min(thickness, 15))}
4. 在独立 threading.Thread 中跑 pipeline.process()
5. 主 asyncio 循环: 等待 thread 完成或客户端断开
6. 返回 FileResponse 或 cancelled
```

**render 使用的 effect_config**: 只传 `body_expand_pixels` 和 `dilate_kernel_size`。未传 `temporal_window`，所以 pipeline 默认值 8 生效。未传 `open_kernel_size` / `blur_sigma`，它们本身也不被消费。

### 8.3 前端 SPA (INDEX_HTML)

三步骤 UI，单文件内嵌 HTML+CSS+JS:

**Step 1** — 上传视频: file input + 上传按钮

**Step 2** — 调参预览:
- 预览图 (base64 即时刷新)
- 人物 ID 复选框 (勾选打码对象)
- 参数控件: fillMode (solid/gradient/blur), opacity (滑块 0-100%), thickness (1-15), fillColor, borderColor
- 300ms 防抖自动刷新预览
- 3秒预览按钮 + 完整视频按钮

**Step 3** — 结果展示: 视频播放器 + 下载链接

**交互机制**:
- 双按钮互斥: 生成预览和渲染不可同时进行
- AbortController: 可通过 `AbortController.abort()` 取消进行中的请求
- 取消按钮: 再次点击进行中的按钮触发取消
- resetToStep1: 三层复位 (网络层 Abort + /cleanup, 数据层清空, UI 层切换面板)

## 12. 两种运行方式对比

| 特性 | CLI (main.py) | Web API (api.py) |
|------|--------------|------------------|
| 入口 | `python main.py -i in.mp4 -o out.mp4` | `uvicorn api:app --port 8002` |
